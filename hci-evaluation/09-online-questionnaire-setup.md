# 在线问卷搭建指南

这个文件用于把 [04-questionnaire.md](04-questionnaire.md) 里的题目搬到问卷星、腾讯问卷或飞书表单中。建议优先用腾讯问卷或问卷星，因为被试只需要打开链接填写，结果可以在平台后台统一导出。

## 推荐方案

推荐使用“线上原型链接 + 线上问卷链接”的方式：

```text
体验网页：
https://bzpony.github.io/WorldView/myMap.html

填写问卷：
发布后把这里替换成你的问卷链接
```

GitHub Pages 只负责展示原型，不负责收集问卷数据。问卷数据建议交给问卷平台保存，最后从平台导出 Excel 或 CSV。

## 平台设置

新建问卷时建议这样设置：

- 问卷类型：普通问卷或调研问卷。
- 收集方式：匿名收集，不要求填写姓名、手机号、学号。
- 登录限制：不强制登录，方便熟人直接填写。
- 每人限填：如果平台支持，可以开启“每个设备或微信限填一次”；如果测试人数很少，也可以不开。
- 提交后提示语：感谢参与，本次测试数据只用于人机交互课程作业分析。
- 结果通知：可以开启“新答卷提醒”，但最终数据以平台后台导出的 Excel/CSV 为准。
- 截止时间：可以不设置，等收满 3 到 5 份后手动停止收集。

不要把个人邮箱直接写进 GitHub 仓库或网页代码里。邮箱提醒可以在问卷平台后台设置。

## 问卷标题

```text
WorldView 人机交互测试后问卷
```

## 问卷说明

```text
感谢参与 WorldView 原型测试。本问卷用于了解你对界面理解、地图与时间轴交互、查找筛选、创建编辑等功能的使用感受。

请先用电脑浏览器体验网页：
https://bzpony.github.io/WorldView/myMap.html

建议先完成这些操作：
1. 自由浏览 1 分钟，判断这个系统是做什么的。
2. 找一个人物并打开详情。
3. 拖动底部时间轴，观察地图变化。
4. 使用筛选功能，只显示人物或地点。
5. 尝试创建一个新地点。
6. 尝试修改地点名称或颜色。

如果某一步不会做，可以跳过，并在开放题中写下卡住的地方。测试的不是你，而是这个界面的易用性。

量表题中：
1 = 非常不同意
2 = 不同意
3 = 一般
4 = 同意
5 = 非常同意
```

## 题目设置

### 一、基本信息

1. 你的编号
   - 题型：单行文本
   - 必填：否
   - 说明：如果是现场主持，可以由主持人提前告诉被试填 P01、P02、P03；如果远程自助，可以不填。

2. 你的测试方式
   - 题型：单选题
   - 必填：是
   - 选项：现场主持 / 线上会议主持 / 远程自助

3. 你使用的设备
   - 题型：单选题
   - 必填：是
   - 选项：电脑 / 平板 / 手机

4. 你是否有世界观、小说、跑团、游戏设定等创作经验
   - 题型：单选题
   - 必填：是
   - 选项：有 / 少量 / 无

5. 你是否常用地图类应用
   - 题型：单选题
   - 必填：是
   - 选项：经常 / 偶尔 / 很少

### 二、整体理解

这一组建议使用“矩阵量表题”，列为 1、2、3、4、5。

1. 我能较快理解这个系统的主要用途。
2. 我能理解系统中“人物、地点、组织、政权”等资源的含义。
3. 页面布局让我知道应该从哪里开始操作。

### 三、地图与时间轴

这一组建议使用“矩阵量表题”，列为 1、2、3、4、5。

4. 地图上的实体标记容易理解。
5. 我能理解时间轴和地图变化之间的关系。
6. 拖动时间轴后的变化足够明显。
7. 运动轨迹和途径点的显示方式容易理解。

### 四、查找与筛选

这一组建议使用“矩阵量表题”，列为 1、2、3、4、5。

8. 我能顺利找到想查看的人物、地点或组织。
9. 侧边栏的资源分类清楚。
10. 搜索功能容易发现和使用。
11. 筛选功能对缩小查找范围有帮助。

### 五、详情、编辑与创建

这一组建议使用“矩阵量表题”，列为 1、2、3、4、5。

12. 详情面板中的信息结构清楚。
13. 我能发现哪些内容可以编辑。
14. 修改名称、颜色等信息的方式容易理解。
15. 创建新地点或新实体的流程清楚。
16. 创建或编辑途径点的流程清楚。

### 六、容错与整体体验

这一组建议使用“矩阵量表题”，列为 1、2、3、4、5。

17. 撤销/重做功能让我觉得操作更安全。
18. 删除实体时的提示让我能理解操作后果。
19. 整体来说，这个系统容易使用。
20. 我愿意用这个工具管理一个架空世界设定。

### 七、开放问题

开放题建议使用“多行文本题”。

1. 你最容易理解的功能是什么？
   - 必填：是

2. 你最困惑或最卡住的地方是什么？
   - 必填：是

3. 有没有哪个按钮、词语、图标或操作方式让你不确定？
   - 必填：是

4. 如果只能优先改进一个地方，你希望改哪里？
   - 必填：是

5. 你觉得这个系统适合什么样的使用场景？
   - 必填：否

6. 你还有其他建议吗？
   - 必填：否

## 快速建表步骤

1. 打开问卷星、腾讯问卷或飞书表单。
2. 新建空白问卷。
3. 填入“问卷标题”和“问卷说明”。
4. 先添加 5 个基本信息题。
5. 添加 5 组矩阵量表题，分别对应 Q1-Q3、Q4-Q7、Q8-Q11、Q12-Q16、Q17-Q20。
6. 添加 6 个开放问题。
7. 预览问卷，自己完整填一遍，确认能提交。
8. 发布问卷，复制问卷链接。
9. 把问卷链接发给被试，或替换到 [08-friend-invite-message.md](08-friend-invite-message.md) 的邀请话术中。
10. 收满 3 到 5 份后，在平台后台导出 Excel/CSV。

## 发给熟人的最终消息模板

发布问卷后，把下面的“问卷链接”替换成平台生成的链接。

```text
你好，我在做人机交互课程项目，需要找几个熟人试用一个网页原型并填问卷，大约 10 到 15 分钟。

先体验网页：
https://bzpony.github.io/WorldView/myMap.html

体验时请大致做这几件事：
1. 自由浏览 1 分钟，看看你觉得它是做什么的。
2. 找一个人物并打开详情。
3. 拖动底部时间轴，观察地图变化。
4. 使用筛选功能，只显示人物或地点。
5. 尝试创建一个新地点。
6. 尝试修改地点名称或颜色。

体验后填写问卷：
问卷链接

如果某一步不会做，可以跳过，但请在问卷里写一下哪里让你困惑。测试的不是你，是这个系统的易用性。
```

## 数据收集后怎么用

导出结果后，建议按 [05-analysis-template.md](05-analysis-template.md) 整理：

- 计算 Q1-Q20 每题平均分。
- 找出平均分低于 3.5 的题，作为优先分析问题。
- 汇总开放题中重复出现的困惑点。
- 把观察记录和问卷结果合并，形成“问题、证据、改进建议”的表格。
