# WorldView 人机交互测试文档

本目录存放 WorldView 课程项目的人机交互测试材料。项目当前已经部署为线上网页原型，可以直接发链接给被试试用，也可以由主持人现场观察被试完成任务。

测试入口：

```text
https://bzpony.github.io/WorldView/
```

项目仍然是课程原型，不是正式发布的 App。因此测试目标不是验证产品是否“完工”，而是观察用户是否能理解并完成核心交互任务。

## 测试定位

测试对象不是代码质量，也不是被试的能力，而是用户在使用原型完成典型任务时的理解、操作和反馈。

被试只需要打开测试链接，不需要查看代码仓库、README 或开发工具。

## 推荐测试方式

优先推荐“主持式任务测试 + 后测问卷”：

- 你把链接发给被试，或现场打开链接。
- 被试按任务操作原型。
- 你记录完成情况、卡点、用时和用户原话。
- 操作结束后，被试填写问卷。
- 最后做 3 到 5 分钟简短追问。

如果时间或场地不方便，也可以采用“远程自助试用 + 问卷”：

- 给被试发送 [07-participant-guide.md](07-participant-guide.md) 中的说明。
- 被试自行打开链接并完成简化任务。
- 被试填写问卷。
- 这种方式更省事，但缺少观察数据，报告中要说明这是远程自助测试。

## 测试前准备

1. 用电脑浏览器打开测试入口：`https://bzpony.github.io/WorldView/`。
2. 确认能看到地图、左侧侧边栏和底部时间轴。
3. 确认网络能加载地图瓦片和 Leaflet CDN。
4. 准备计时工具、观察记录表和问卷。
5. 每位被试测试前刷新页面，保证初始状态一致。

## 推荐试用人数

第一轮小范围试用推荐 3 到 5 人。先找 1 个熟人做 pilot test，确认链接、任务和问卷都没问题，再找剩下的人正式试用。

- 优先找熟悉网页操作、愿意认真反馈的人。
- 如果能找到喜欢写作、跑团、游戏设定或世界观创作的人更好。
- 不需要做正式招募，也不需要追求样本很大；课程实践重点是发现典型可用性问题。

如果时间紧，至少完成 1 人 pilot test + 3 人正式试用。

## 建议输出成果

课程报告中建议包含：

- 测试目标和方法
- 被试基本信息
- 任务完成率和平均用时
- 关键问题列表
- 问卷统计结果
- 用户原话摘录
- 后续改进方案
