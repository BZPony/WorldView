# WorldView 人机交互测试文档

本目录存放 WorldView 课程项目的人机交互测试材料。项目目前是一个代码仓库中的前端原型，不是已经打包发布的正式 App，因此测试方式应定位为“原型可用性测试”，而不是产品验收测试。

## 测试定位

测试对象不是代码质量，也不是被试的技术能力，而是用户在使用原型完成典型任务时的理解、操作和反馈。

测试时建议由主持人提前打开 `myMap.html`，让被试只看到浏览器中的界面。被试不需要看 README、代码目录或开发工具。

## 推荐测试方式

建议采用“任务测试 + 观察记录 + 问卷 + 简短访谈”的组合。

单纯问卷只能得到主观满意度，无法判断用户到底卡在哪里。任务测试可以观察真实操作过程，问卷和访谈再补充用户的主观感受。

## 测试前准备

1. 确认电脑能联网，因为 Leaflet 和地图瓦片依赖 CDN。
2. 用浏览器打开项目根目录下的 `myMap.html`。
3. 关闭或隐藏代码编辑器，避免被试误以为需要理解代码。
4. 准备计时工具、观察记录表和问卷。
5. 每位被试测试前刷新页面，保证初始状态一致。

## 推荐被试数量

第一轮测试推荐 5 到 8 人。

- 目标用户：喜欢世界观设定、写作、跑团、游戏设定或地图创作的人，3 到 5 人。
- 普通用户：熟悉网页操作但没有明显世界观创作经验的人，2 到 3 人。

## 建议输出成果

课程报告中建议包含：

- 测试目标和方法
- 被试基本信息
- 任务完成率和平均用时
- 关键问题列表
- 问卷统计结果
- 用户原话摘录
- 后续改进方案
