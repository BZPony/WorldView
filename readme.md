# WorldView — 架空世界时空可视化工具

前排提示：此软件是一个课程作业，代码乱到我自己都看不下去，请不要抱有过多期望=(

欢迎来到 WorldView！这是一款面向架空世界创作者的**交互式地图 + 时间轴可视化工具**。你可以在真实世界地图上绘制人物迁徙、政权更迭、组织兴衰，沿着时间轴观看世界如何随岁月流转。

无需安装任何软件，浏览器打开即用。本手册将通过一系列**情景教学**带你从零开始掌握所有操作。如果你是开发者，请直接跳转到[面向开发者的架构文档](#面向开发者)。

## 快速开始

1. 用浏览器打开项目目录下的 `myMap.html` 文件（需要网络连接，地图瓦片和 Leaflet 库通过 CDN 加载）。
2. 页面加载完成后，你将看到一张世界地图，左侧有侧边栏，底部有时间轴。
3. 开始探索吧！

> **提示**：本项目纯前端运行，无需 Node.js、无需后端服务、无需 `npm install`。

## 界面总览

打开后的界面由四个主要区域组成：

```
┌──────────────────────────────────────────────────────┐
│    [侧边栏]     [详情]              [地图区域]          │
│  ┌────────┐┌──────────┐                              │
│  │        ││          │                              │
│  │        ││          │                              │
│  │        ││          │                              │
│  │        ││          │                              │
│  │        ││          │                              │
│  └────────┘└──────────┘  ▓▓▓▓▓▓▓▓▓ [时间轴] ▓▓▓▓▓▓▓▓  │
└──────────────────────────────────────────────────────┘
```

| 区域 | 用途 |
|------|------|
| **侧边栏** | 左侧面板，包含"筛选"和"资源"两个 Tab。筛选用于按条件过滤实体，资源管理器按类别列出所有实体。 |
| **详情面板** | 侧边栏右侧弹出的面板，显示当前选中实体的所有信息，支持直接单击编辑。关闭按钮（✕）、图钉按钮（📌，仅对有轨迹的实体显示）、删除按钮（红色）。 |
| **地图** | 中央区域，显示实体的彩色标记（圆形图标 + 名称标签）和运动轨迹线。可拖拽、缩放。 |
| **时间轴** | 底部横向滚动条，红色竖线代表"当前时间"。拖动时间轴可以让世界"穿越"到任意时刻。右侧有缩放按钮。 |

## 情景教学

### 场景一：浏览世界地图

🚩 **目标**：熟悉地图基本操作，找到感兴趣的实体。

1. **平移地图**：用鼠标拖拽地图任意位置。
2. **缩放地图**：使用鼠标滚轮，或点击地图右上角的 `+` `-` 按钮。
3. **识别实体**：地图上的圆形图标代表一个实体（人物、地点、组织、政权）。每个图标右侧有该实体的名称标签，颜色与实体主题色一致。
4. **轨迹细线**：连接同一实体多个途径点的轨迹线。线条上的小圆点是该实体曾经停留过的位置。
5. **侧边栏开关**：点击左侧面板顶部的 < 按钮可关闭侧边栏，让地图获得更多空间。关闭后，左上角角会出现一个菜单按钮，点击可重新打开。

---

### 场景二：沿着时间旅行

🚩 **目标**：理解时间轴，观察实体如何随时间变化。

1. **拖动时间光标**：在底部时间轴上按住鼠标左键拖动。你会看到：
   - 地图上实体的位置随光标移动而改变（如果该实体有运动轨迹）。
   - 轨迹线上的途径点标记会在光标经过时高亮显示。
   - 实体名称随名称演变（nameHistory）自动更新。
2. **理解刻度**：时间轴上的刻度线代表年份。默认缩放级别为「年」。
3. **缩放时间精度**：点击时间轴左侧的 `🔍+` / `🔍-` 缩放按钮，可以在五种精度之间切换：**日 → 月 → 年 → 十年 → 百年**。放大到月/日级别可以观测更精细的时间变化。

> **提示**：实体的途径点有一个 `resolution`（分辨率）属性。例如一个 `resolution: "year"` 的途径点，在"月"级别缩放时才能在地图上看到。这避免了低精度数据在高精度视图中产生杂乱。

1. **Lifespan Bars**：时间轴刻度上方如果有彩色横条，代表某个实体的"活跃时间范围"。人物实体显示为带虚线边框的彩色条（虚线部分表示出生-死亡区间），普通实体显示为实心彩色条。这些条通过"图钉"功能控制显示（见场景八）。

---

### 场景三：查看实体详情

🚩 **目标**：了解如何查看和读取一个实体的完整信息。

1. **点击地图标记**：点击地图上任意实体的圆形图标。会发生三件事：
   - 该图标的外环变为**蓝色高亮发光**状态。
   - 左侧弹出**详情面板**，展示该实体的所有组件信息。
   - 侧边栏自动切换到"资源"Tab，展开对应分类并滚动到该实体。
2. **点击侧边栏列表项**：你也可以在侧边栏的"资源"Tab 中点击任意实体名称，效果与点击地图标记相同。
3. **关闭详情面板**：点击详情面板右上角的 ✕ 按钮。再次点击任意实体可重新打开。

**详情面板内容解读：**

详情面板按组件类型分组显示信息。常见的组件有：

| 组件 | 显示内容 | 含义 |
|------|---------|------|
| **core** | 名称、颜色、图标 | 实体的基本标识 |
| **person** | 出生/死亡时间、性别、描述 | 人物专属信息 |
| **place** | 经纬度坐标、描述 | 固定地点的位置 |
| **motion** | 途径点列表 | 实体在地图上的运动路径 |
| **nameHistory** | 名称演变记录 | 实体在不同时间段的名称 |
| **organization** | 总部、领袖、成员 | 组织架构信息 |
| **regime** | 首都、人口、政体 | 政权属性 |
| **customTags** | 自定义标签 | 用户分类标记 |

### 场景四：在侧边栏中查找资源

🚩 **目标**：使用侧边栏的资源管理器快速定位实体。

1. 确保侧边栏处于打开状态（左侧有面板）。
2. 点击侧边栏顶部的 **"资源"** Tab 按钮。
3. 你会看到按类型分组的面板：**人物、地点、组织、政权** 以及自定义标签分组。
4. **展开/折叠**：点击分类标题（如"👤 人物"）可以展开或收起该类别的实体列表。
5. **选中实体**：点击列表中的实体名称，地图标记将高亮，详情面板将打开。
6. **右键删除**：在列表项上**右键**可以弹出删除菜单。

> **提示**：展开/折叠状态会在列表刷新后保持（例如筛选条件变化时）。

---

### 场景五：创建新实体

🚩 **目标**：在架空世界中添加一个新人物、地点、组织或政权。

所有创建操作通过**右键菜单**完成：

1. **在地图空白处右键**，弹出上下文菜单。
2. 选择要创建的实体类型：
   - **创建 Person**（人物）
   - **创建 Place**（地点）
   - **创建 Organization**（组织）
   - **创建 Regime**（政权）
3. 系统会自动在点击位置创建一个新的实体（名称默认为"新人物"/"新地点"等）。
4. 详情面板自动打开，你可以立即编辑名称、颜色、图标等信息（见场景六）。
5. 如果创建的是 Place，地图上会出现该地点的标记。

**为已有实体添加途径点：**

1. 在地图空白处右键 → 选择 **创建 Waypoint**。
2. 会弹出实体选择列表，选择一个已有 motion 实体会在该位置添加一个途径点。
3. 途径点会自动按时间排序插入，副面板自动打开供你编辑详情。

---

### 场景六：编辑实体信息

🚩 **目标**：修改实体的名称、颜色、图标及其他属性。

1. **打开详情面板**：点击地图标记或侧边栏列表项。
2. **单击属性值**：在详情面板中，直接**单击**任意属性值区域（如名称、颜色、描述等）。该区域将变为可编辑状态。
3. **修改文本字段**：直接输入新内容，按 Enter 或点击面板外区域确认。
4. **修改颜色**：单击颜色属性值，会出现一个内联的**颜色选择器**（浏览器原生取色器）。选择新颜色后自动保存，地图标记颜色同步更新。
5. **修改图标**：单击图标属性值，会弹出一个**图标选择器弹窗**。按类别（实体/系统/操作）浏览，点击目标图标即可替换。按 ESC 或点击遮罩层关闭弹窗。
6. **修改时间字段**：person 组件的出生/死亡时间、motion 途径点的到达/离开时间均支持编辑（单击后输入年月日）。

> **所有修改自动保存，并支持撤销（Ctrl+Z）**。

### 场景七：管理运动轨迹

🚩 **目标**：为人物添加、编辑、删除途径点，构建复杂路线。

**查看途径点：**

1. 选中一个有 motion 组件的实体（如示例中的"杰洛特"）。
2. 详情面板中展开 **motion** 组件，可以看到途径点列表。每个途径点显示名称、描述、到达/离开时间、位置等信息。

**编辑途径点位置（Location Picker）：**

1. 在途径点条目中，点击位置旁的 **"定位"** 按钮。
2. 地图进入定位选取模式（光标会变化）。
3. 此时你可以：
   - **点击已有的地点实体** → 途径点绑定到该地点（`type: 'place'`）。此后如果该地点位置改变，途径点会自动跟随移动。
   - **点击地图空白处** → 创建坐标位置（`type: 'coords'`）。
4. 按 **ESC** 键退出定位模式。

**编辑途径点时间 / 名称 / 描述：**

- 直接在详情面板中单击对应字段编辑，与普通属性编辑方式相同。

**添加途径点：**

- 在地图任意位置右键 → 选择 **创建 Waypoint** → 选择目标实体 → 系统自动插入到时间排序的正确位置。

**删除途径点：**

- 详情面板中每个途径点条目有一个删除按钮，点击即可删除该途径点。

**副面板（Secondary Panel）：**

- 创建 Waypoint 或点击途径点的某些操作后，会自动打开**第二个详情面板**（副面板），用于同时编辑途径点详情。副面板独立于主面板，可以同时存在。

### 场景八：使用时间轴高级功能

🚩 **目标**：掌握 Lifespan Bars 和 Thumbtack 图钉功能。

**Lifespan Bars（生命条）：**

- 时间轴刻度上方的彩色横条表示实体的活跃时间范围。
- **Person 实体**：显示虚线边框的彩色条。出生前和死亡后的活动区间用虚线表示，出生-死亡区间两端有 🍼 和 💀 图标标记。
- **Motion 实体**：显示实心彩色条，覆盖从第一个途径点到最后一个途径点的时间范围。
- 多个条带从底部向上堆叠排列，颜色与实体主题色一致，带有白色轮廓增强对比。

**Thumbtack 图钉：**

- Thumbtack 用于**控制哪些实体的 Lifespan Bar 显示在时间轴上**。
- 选中一个有 motion 组件的实体，详情面板右上角会出现一个 📌 **图钉按钮**。
- 点击图钉按钮，该实体的 lifespan bar 出现在时间轴上。再次点击取消固定。
- 可以同时固定多个实体，它们的 lifespan bars 会同时显示。
- 如果实体没有 motion 组件，图钉按钮会自动隐藏。

### 场景九：使用筛选缩小范围

🚩 **目标**：当实体数量较多时，使用筛选快速找到目标。

1. 点击侧边栏顶部的 **"筛选"** Tab 按钮。
2. 筛选面板包含以下筛选器：

| 筛选器 | 操作 | 效果 |
|--------|------|------|
| **搜索框** | 输入文字 | 实时模糊匹配实体名称（不区分大小写） |
| **实体类型** | 勾选/取消复选框 | 只显示指定类型的实体。支持"全选"/"取消全选"快捷按钮 |
| **时间筛选** | 打开开关 → 选择"跟随时间轴"或"自定义时间" | 只显示当前时间窗口内有活动的实体。自定义时间可输入起止年月 |
| **地图范围** | 打开开关 | 只显示当前地图视野内的实体。拖拽或缩放地图时自动更新 |
| **重置按钮** | 点击"重置所有筛选" | 恢复所有筛选器为默认值（显示全部） |

1. 筛选 Tab 按钮上会出现一个**蓝色小圆点 badge**，显示当前激活的筛选条件数量。
2. 筛选仅影响侧边栏的资源列表显示，不会修改或删除任何实体数据。
3. 取消筛选即可恢复完整列表。

### 场景十：撤销与重做

🚩 **目标**：操作出错时回退到之前的状态。

WorldView 支持完整的撤销/重做，覆盖所有数据修改操作（编辑、创建、删除）。

| 操作 | 快捷键 |
|------|--------|
| **撤销** | `Ctrl + Z`（Windows）/ `Cmd + Z`（Mac） |
| **重做** | `Ctrl + Shift + Z`（Windows）/ `Cmd + Shift + Z`（Mac） |

你可以连续撤销多步操作，直到回到初始状态；也可以重做之前撤销的操作。撤销/重做不会影响地图视图的位置和缩放。

### 场景十一：删除实体

🚩 **目标**：移除不再需要的实体。

**方法一：详情面板删除**

1. 选中目标实体，打开详情面板。
2. 点击右上角的**红色删除按钮**。
3. 在弹出的确认弹窗中点击确认。
4. 该实体从地图、侧边栏和数据库中移除。详情面板自动关闭。

**方法二：侧边栏删除**

1. 在侧边栏"资源"Tab 中找到目标实体。
2. **右键**点击该实体名称 → 选择"删除"。
3. 如果删除的正是当前选中的实体，详情面板会自动关闭。

> **提示**：删除操作完全支持撤销（Ctrl+Z），误删可以立即恢复。

## 面向开发者

以下内容面向希望二次开发、扩展功能或理解系统架构的开发者。普通用户可以跳过此部分。

### ECS 数据架构

WorldView 采用 **Entity-Component-System**（实体-组件-系统）模式。每个实体只是一个 ID + 一组可动态挂载的组件：

| 组件 | 类型标识 | 含义 |
|------|---------|------|
| **core** | `core` | 必备基础组件（名称、颜色、图标） |
| **person** | `person` | 人物信息（出生/死亡、性别） |
| **place** | `place` | 固定位置（坐标、描述） |
| **motion** | `motion` | 运动轨迹（途径点数组） |
| **nameHistory** | `nameHistory` | 名称演变（时间段→名称） |
| **organization** | `organization` | 组织信息（总部、领袖、成员） |
| **regime** | `regime` | 政权信息（首都、人口、政体） |
| **customTags** | `customTags` | 自定义标签数组 |

数据定义在 `js/data/entities.js`，通过工厂函数创建。修改实体数据的唯一入口是 `CommandHandler`（命令模式），确保所有修改可追踪、可撤销。

### 事件驱动 + 状态管理

- **EventBus**：发布/订阅事件总线。模块间通信通过事件，不直接调用。常见事件：`ui:select`、`layout:change`、`state:change`、`command:execute`。
- **AppState**：全局单一数据源。读取：`AppState.get('key')`；写入：`AppState.set('key', value)`。写入时自动广播 `state:change` 事件。
- **CommandHandler**：命令处理器。所有数据修改打包为命令执行，维护 undo/redo 双栈。快照采用 `entities` 数组的深拷贝。

### 如何扩展

**添加新的实体类型（如事件、物品）：**

1. 在 `entities.js` 中定义组件工厂函数和实体工厂函数。
2. 在 `sidebar.js` 的 `typeConfig` 中添加图标和标签。
3. 在 `js/components/detail/componentRenderers/` 中创建渲染器。
4. 在 `svg-icons.js` 中添加 SVG 图标。
5. 在 `contextMenu.js` 中添加创建菜单项。
6. 在 `commandHandler.js` 中添加对应的创建命令。

**添加新的左侧/右侧面板：**

- 创建 CSS（复用 `.panel--left` 基类）。
- 创建 JS 模块，管理 DOM、事件和状态。
- 在 `LayoutManager.panels` 中注册，时间轴会自动适配。

### 依赖与调试

- **Leaflet 1.7.1**（CDN 引入）
- 纯原生 JavaScript，无框架依赖
- 图标为内联 SVG，无外部图标库

**调试技巧：**

- `window.EventBus` — 浏览器控制台可查看所有事件订阅 `EventBus.events`
- `window.AppState` — 查看/修改全局状态 `AppState.get('key')`
- 监听 `state:change` 事件可追踪所有状态变更流
